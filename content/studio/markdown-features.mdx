# Markdown Features

Markdown gives each lesson a simple writing rhythm: plain text first, structure only where it helps. Use these built-in patterns for examples, notes, grammar tables, reading practice, and reference links.

## Headings

Use headings to break a long lesson into scan-friendly steps.

```md
## Sound Practice
### Short Vowels
```

## Paragraphs And Emphasis

Write normal paragraphs for explanation. Use **bold** for important words and *italic* for pronunciation hints.

```md
The word **नमस्ते** is often read as *namaste*.
```

The word **नमस्ते** is often read as *namaste*.

## Lists

Use unordered lists for quick groups.

- Listen once.
- Repeat slowly.
- Write the word from memory.

Use numbered lists when order matters.

1. Read the word.
2. Cover the answer.
3. Say it aloud.

## Blockquotes

Use `>` for a quoted rule, a reminder, or a sentence worth keeping visible.

```md
> A short vowel and a long vowel can change the meaning.
```

> A short vowel and a long vowel can change the meaning.

## Tables

Tables are useful for vocabulary, scripts, meanings, and short grammar contrasts.

```md
| Sanskrit | Transliteration | Meaning |
| --- | --- | --- |
| फलम् | ==phalam== | fruit |
| जलम् | jalam | water |
```

| Sanskrit | Transliteration | Meaning |
| --- | --- | --- |
| फलम् | ==phalam== | fruit |
| जलम् | jalam | water |

Use `tableCell` when one table cell needs a main value and a smaller helper line below it. The helper line appears in muted italic text inside parentheses.

```md
| Group | Example |
| --- | --- |
| :tableCell[अकारान्तः पुल्लिङ्गशब्दः]{sub="akārāntaḥ pulliṅgaśabdaḥ"} | :tableCell[बालकः]{sub="boy"} |
```

| Group | Example |
| --- | --- |
| :tableCell[अकारान्तः पुल्लिङ्गशब्दः]{sub="akārāntaḥ pulliṅgaśabdaḥ"} | :tableCell[बालकः]{sub="boy"} |

## Code And Inline Text

Use inline code for syntax examples like `[[answer]]`, `:BookOpen:`, or `:::note`.

For longer examples, use fenced code blocks:

```md
:::tip[Practice Pattern]
Read the sentence once, then reveal the missing word.
:::
```

## Links

Use regular Markdown links for references. External links open in a new tab.

```md
[Lucide icons](https://lucide.dev/icons)
```

[Lucide icons](https://lucide.dev/icons)

## Horizontal Rules

Use `---` to create a visual pause between practice sections.

---

## Explore Blocks

Use `:::explore` when an article needs a small list of external references at the end.

```md
:::explore
[>] [Markdown Guide](https://www.markdownguide.org/basic-syntax/)  - Basic Syntax
[>] [GitHub Flavored Markdown Spec](https://github.github.com/gfm/)
[>] [CommonMark Help](https://commonmark.org/help/)
:::
```
